# Security Policy

## Supported Versions

Поддерживаются следующие версии с обновлениями безопасности:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Если вы обнаружили уязвимость безопасности:

### ⚠️ НЕ создавайте публичный Issue!

Вместо этого:

1. **Используйте GitHub Security Advisories**
   - Перейдите в раздел Security → Advisories
   - Нажмите "Report a vulnerability"
   - Заполните форму с описанием проблемы

2. **Или свяжитесь напрямую**
   - Email: [ваш email для security отчетов]
   - Укажите детали уязвимости
   - Предложите возможное решение (если есть)

### Что включить в отчет

- **Описание уязвимости**
- **Шаги воспроизведения**
- **Потенциальное влияние**
- **Предложенное исправление** (опционально)
- **Версия ПО** где обнаружена проблема

### Процесс обработки

1. **Подтверждение** - в течение 48 часов
2. **Оценка** - анализ серьезности (1-7 дней)
3. **Исправление** - разработка патча (зависит от сложности)
4. **Релиз** - публикация обновления
5. **Disclosure** - публичное раскрытие после патча

### Типы уязвимостей

Приоритетные:
- Утечка токенов/ключей
- Удаленное выполнение кода
- Повышение привилегий
- SQL инъекции
- XSS в веб-интерфейсах

Низкоприоритетные:
- DoS локального характера
- Информационные утечки низкой критичности

## Безопасные практики

### Для пользователей

- Храните токены в безопасности
- Регулярно обновляйте приложение
- Используйте сильные пароли для Telegram
- Проверяйте SHA256 checksum релизов

### Для разработчиков

- Никогда не коммитьте секреты
- Используйте `appsettings.template.json`
- Проверяйте зависимости на уязвимости
- Включайте CodeQL в CI/CD

- Файле ACKNOWLEDGMENTS.md (если создан)
